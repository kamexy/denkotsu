# 第二種電気工事士 学習アプリ「デンコツ」仕様書

## コンセプト

**「勉強しない勉強法」— スマホを触るだけで、いつの間にか合格できる**

従来の学習アプリの失敗パターンを根本から排除する設計思想:

| 従来のアプリ | デンコツ |
|---|---|
| 毎日の学習ノルマがある | ノルマなし。触りたい時だけ触る |
| スケジュールを立てさせる | アプリが全自動で最適化 |
| テキストを読ませる | 1問10秒のクイズで覚える |
| 連続記録（ストリーク）で煽る | 煽らない。開いたら褒める |
| 全範囲を網羅的に学習 | AI が弱点だけを狙い撃ち |

## 画面構成・コア機能

### 1. ホーム画面 — 「今日のひとくち」

アプリを開いた瞬間、即座に学習が始まる。メニューもチュートリアルもなし。

```
┌─────────────────────────────┐
│                             │
│   ⚡ 合格力 42%             │
│   ████████░░░░░░░░░░░░      │
│                             │
│  ┌─────────────────────┐    │
│  │                     │    │
│  │   [写真: VVFケーブル] │    │
│  │                     │    │
│  │  Q. この電線の名称は？│    │
│  │                     │    │
│  │  ① VVFケーブル       │    │
│  │  ② VVRケーブル       │    │
│  │  ③ IV電線           │    │
│  │  ④ EM-EEFケーブル    │    │
│  │                     │    │
│  └─────────────────────┘    │
│                             │
│  [もう1問] [今日はここまで]   │
│                             │
│  ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─   │
│  📊 学習  🏆 実績  ⚙ 設定  │
└─────────────────────────────┘
```

**設計のポイント:**
- アプリ起動 → 0.5秒以内に問題が表示される
- 「もう1問」ボタンで次々解ける（TikTok的な中毒性）
- 「今日はここまで」に罪悪感を感じさせない（「お疲れさま！また来てね」）
- 合格力ゲージが唯一のKPI（日付ベースの進捗は見せない）

### 2. クイズエンジン — 心理的負荷ゼロの学習

#### 出題形式（すべて10〜30秒で回答可能）

**① 4択クイズ（メイン）**
- 過去問ベース
- 写真・図面付き
- 正解後に「ひとこと解説」（3行以内）

**② ◯✕クイズ**
- 法規・安全に関する知識
- スワイプで回答（右=◯、左=✕）

**③ 画像タップ**
- 配線図・器具写真から正しい部分をタップ
- 「この記号の器具はどれ？」→ 4枚の写真から選ぶ

**④ 並べ替え**
- 作業手順の正しい順番に並べる
- ドラッグ&ドロップで直感的に操作

**⑤ 穴埋めフラッシュ**
- 「許容電流が30Aの電線の太さは ___mm²」
- 数値をスライダーで選択

#### 適応学習アルゴリズム

```
出題優先度 = 忘却曲線スコア × 重要度 × 苦手度

- 忘却曲線スコア: 最後に正解してからの経過時間に基づく
- 重要度: 過去10年の出題頻度から算出
- 苦手度: 過去の正答率から算出（間違えた問題は高頻度で再出題）
```

**ポイント:**
- ユーザーは何も考えなくていい
- 「今日はこの分野をやろう」という判断が不要
- アプリが最も効率的な順番で自動出題

### 3. 合格力メーター

試験日や学習計画は一切見せない。代わりに「合格力」という単一指標。

```
⚡ 合格力 42%

内訳:
├── 電気理論    ██████░░░░ 60%
├── 配線図      ████░░░░░░ 40%
├── 法規        ███░░░░░░░ 30%
├── 工事方法    █████░░░░░ 50%
├── 器具・材料  ██████░░░░ 55%
└── 検査・測定  ████░░░░░░ 35%
```

- 全分野が70%以上 → 合格力100% → 「合格圏内！」表示
- 各分野をタップすると、その分野だけ集中的に解ける
- 数値は過去問の正答率×忘却曲線を加味した「今解いたら何%取れるか」の推定値

### 4. ごほうびシステム

学習のモチベーション維持を「習慣」ではなく「報酬」で実現。

#### コレクション：電気工事パーツ図鑑

問題を解くとパーツがランダムでドロップ:
- ⚡ ブレーカー（ノーマル）
- ⚡ VVFケーブル（ノーマル）
- 🌟 ホーロー碍子（レア）— 昔の電気工事で使われた器具
- 💎 碍子引き工事セット（レジェンド）— 今はほぼ見ない工法

各パーツに豆知識が付いており、コレクション自体が学習コンテンツになる。

#### 実績（アチーブメント）

- 🔰 はじめの一歩: 初めて問題を解いた
- ⚡ 10問連続正解: 連続正解記録
- 🧠 電気理論マスター: 電気理論の合格力80%達成
- 🏆 全分野制覇: すべての分野で合格力70%以上
- 🎯 過去問キラー: 過去問正答率90%以上
- 🌙 夜型電気工事士: 23時以降に100問解いた
- 🚃 通勤マスター: 平日7-9時に50問解いた

### 5. 技能試験サポート（フェーズ2）

筆記試験合格後に開放される技能試験対策コンテンツ。

- 候補問題13問の施工手順を30秒〜1分のショート動画で解説
- 各作業（ケーブル切断、被覆剥ぎ、圧着等）のコツを写真付きカードで
- 複線図の描き方を段階的アニメーションで
- 欠陥判定クイズ: 「この施工は合格？不合格？」

## UX設計原則

### 1. 「0秒起動」

- PWAでホーム画面にインストール
- オフライン対応（電波なしでも学習可能）
- 起動後、前回の続きから即座に問題表示
- ログインは後回し（ゲスト利用→後からアカウント連携）

### 2. 「罪悪感ゼロ」

- ストリーク（連続日数）を表示しない
- 「今日はまだ勉強していません」的な通知を送らない
- 久しぶりに開いても「おかえり！」だけ
- 通知は「週1回のやさしいリマインダー」のみ（オフ可）

### 3. 「1分で完結」

- 1問あたり10〜30秒
- 5問解けば1セット完了
- 「トイレ1回 = 1セット」がちょうどいい

### 4. 「SNS的中毒性」

- 問題を解くたびに気持ちいいアニメーション
- 正解時のサウンドエフェクト（バイブレーション対応）
- 合格力メーターが少しずつ上がる達成感
- コレクションのドロップ演出

## 技術スタック — 「0円スタート」アーキテクチャ

### 設計思想: クライアントファースト

```
従来の設計:
  ブラウザ → サーバー → DB → サーバー → ブラウザ
  (サーバー代がかかる)

デンコツの設計:
  ブラウザ → IndexedDB（ローカル）
  (サーバー代ゼロ。クラウド同期はオプション)
```

**核心: 学習アプリに常時接続のサーバーは不要。**
問題データは静的ファイル、学習履歴はブラウザ内DB、適応学習アルゴリズムもクライアント側で実行。
サーバーが必要になるのは「クラウド同期」と「課金」だけ。それも無料枠で十分収まる。

### Phase別の技術スタックと月額コスト

#### Phase 1: MVP — 月額0円

| 要素 | 技術 | コスト |
|---|---|---|
| フレームワーク | **Next.js 15** (Static Export) | — |
| 言語 | **TypeScript** | — |
| CSS | **Tailwind CSS v4** | — |
| アニメーション | **Framer Motion** | — |
| ホスティング | **Cloudflare Pages** | 無料（帯域無制限） |
| 問題データ | **静的JSON** (ビルド時に生成) | 無料（CDNから配信） |
| ユーザーデータ | **IndexedDB** (ブラウザ内DB) | 無料（サーバー不要） |
| 適応学習 | **クライアントサイドJS** | 無料（サーバー不要） |
| PWA/オフライン | **Service Worker** + **Cache API** | 無料 |
| 分析 | **Cloudflare Web Analytics** | 無料 |
| ドメイン | denkotsu.com 等 | 約1,500円/年 |
| **合計** | | **約125円/月** (ドメインのみ) |

この構成で **数万ユーザーまでサーバー代0円** で対応可能。

```
アプリ構成:
┌─────────────────────────────────────────────┐
│ Cloudflare Pages (無料)                      │
│ ├── /index.html          (SPA)              │
│ ├── /data/questions.json (問題データ)        │
│ ├── /images/             (器具写真等)        │
│ ├── /sw.js               (Service Worker)   │
│ └── /manifest.json       (PWA設定)          │
└─────────────────────────────────────────────┘
         ↕ (すべてCDN経由、サーバーレス)
┌─────────────────────────────────────────────┐
│ ユーザーのブラウザ                            │
│ ├── IndexedDB: 学習履歴・進捗・設定          │
│ ├── Cache API: 問題データのオフラインキャッシュ│
│ └── JS: 忘却曲線計算・出題アルゴリズム       │
└─────────────────────────────────────────────┘
```

**IndexedDB で管理するデータ:**
- 各問題の回答履歴（正誤、回答時間）
- 忘却曲線パラメータ（ease_factor, interval, next_review）
- 合格力スコア（分野別）
- コレクションアイテム
- ユーザー設定（サウンドON/OFF等）

**トレードオフ:**
- ✅ サーバー代完全ゼロ
- ✅ オフライン完全対応（電波なしでも使える）
- ✅ 超高速（ネットワーク往復なし）
- ⚠️ 端末を変えるとデータが引き継げない（Phase 2で解決）
- ⚠️ ブラウザのデータ削除で学習履歴が消える（Phase 2で解決）

#### Phase 2: クラウド同期追加 — 月額0円（無料枠内）

ユーザーが増えてきたら、端末間同期とアカウント機能を追加。

| 要素 | 技術 | 無料枠 |
|---|---|---|
| 認証 | **Cloudflare Zero Trust** or **Supabase Auth** | 50,000 MAU無料 |
| データ同期 | **Cloudflare D1** (SQLite at Edge) | 読取5M/日、書込100K/日 無料 |
| OR | **Supabase** (PostgreSQL) | 500MB DB、50K MAU 無料 |
| 画像ストレージ | **Cloudflare R2** | 10GB無料、転送料0円 |
| **合計** | | **0円**（無料枠内） |

**Cloudflare D1 が最強の理由:**
- SQLiteベースなのでシンプル
- Edge（CDN拠点）で実行されるため超高速
- 読取5M回/日 = 1ユーザー50回/日として、10万ユーザーまで無料
- 書込100K回/日 = 1ユーザー10回/日として、1万ユーザーまで無料
- 転送料0円（Cloudflareの大きな利点）

**同期の仕組み:**
```
ブラウザ (IndexedDB)  ←→  Cloudflare D1 (クラウド)
       ↑                        ↑
  常にこちらが主     ログイン時のみ同期
  （オフライン最優先）
```
- ローカルIndexedDBが常にマスター（オフラインファースト）
- ログインしているユーザーのみ、操作時にバックグラウンド同期
- コンフリクト解決: タイムスタンプベースのLast-Write-Wins

#### Phase 3: 課金追加 — 月額0〜3,000円

| 要素 | 技術 | コスト |
|---|---|---|
| 決済 | **Stripe** | 3.6% 手数料のみ（固定費0円） |
| 広告 | **Google AdSense** (Web) | 収益（コストではない） |
| メール | **Resend** | 月3,000通無料 |
| **合計** | | **0円**（売上の3.6%が手数料） |

| 観点 | Cloudflare統一 | Vercel + Supabase |
|---|---|---|
| 月額コスト（小規模） | 0円 | 0円 |
| 月額コスト（1万ユーザー） | 0円 | 0円 |

## データモデル

### 問題テーブル (questions)
```sql
id: UUID
category: TEXT          -- 電気理論/配線図/法規/工事方法/器具材料/検査測定
subcategory: TEXT       -- 細分類
type: TEXT              -- multiple_choice/true_false/image_tap/sort/fill
difficulty: INT (1-5)
question_text: TEXT
question_image_url: TEXT
options: JSONB          -- 選択肢
correct_answer: TEXT
explanation: TEXT        -- ひとこと解説
explanation_image_url: TEXT
exam_year: INT          -- 出典年度（過去問の場合）
exam_session: TEXT      -- 上期/下期
importance: FLOAT       -- 出題頻度に基づく重要度
created_at: TIMESTAMP
```

### ユーザー学習履歴 (user_progress)
```sql
id: UUID
user_id: UUID
question_id: UUID
is_correct: BOOLEAN
answered_at: TIMESTAMP
time_spent_ms: INT      -- 回答にかかった時間
```

### 忘却曲線データ (spaced_repetition)
```sql
id: UUID
user_id: UUID
question_id: UUID
ease_factor: FLOAT      -- 難易度係数
interval_days: FLOAT    -- 次回出題までの間隔
next_review_at: TIMESTAMP
repetition_count: INT   -- 復習回数
```

### コレクション (user_collections)
```sql
id: UUID
user_id: UUID
item_id: UUID
obtained_at: TIMESTAMP
```

### コレクションアイテム (collection_items)
```sql
id: UUID
name: TEXT
description: TEXT       -- 豆知識
image_url: TEXT
rarity: TEXT            -- normal/rare/legendary
category: TEXT          -- 器具/材料/工具/工法
drop_rate: FLOAT
```

## コンテンツ更新戦略 — 法改正・試験変更への追従

### なぜ更新が必要か

第二種電気工事士試験のコンテンツは「固定」ではない。以下の変更が常に発生する:

| 変更の種類 | 頻度 | 影響度 | 例 |
|---|---|---|---|
| 電気設備技術基準の改正 | 年1〜3回 | 高 | リチウムイオン蓄電池の安全基準追加、EV急速充電設備の保安要件追加 |
| 電気工事士法・施行令の改正 | 数年に1回 | 高 | 学科試験免除期間の拡大（次回→次々回）、受験手数料の改定 |
| JIS規格の改定 | 年数回 | 中 | JIS C 8715-2（蓄電池安全規格）の引用追加 |
| 試験制度の変更 | 年1回程度 | 中 | CBT実施期間の延長（18日→46日）、申込のインターネット原則化 |
| 技能試験候補問題の更新 | 毎年1月 | 高 | 13問の候補問題が入れ替わる可能性 |
| 新しい過去問の公開 | 年2回 | 中 | 上期・下期試験終了後に問題と正答が公開 |
| 新材料・新工法の登場 | 不定期 | 低 | 新しいケーブル規格、施工方法の追加 |

### 更新体制（3層構造）

#### 第1層: 自動監視（コスト: ゼロ〜低）

```
┌─────────────────────────────────────────┐
│           自動監視パイプライン            │
│                                         │
│  ① Webスクレイピング/RSSフィード         │
│     - 電気技術者試験センター公式サイト    │
│     - 経済産業省 産業保安グループ        │
│     - e-Gov法令検索 API                  │
│                                         │
│  ② 差分検知 → Slack/メール通知          │
│     - ページ変更を検出                   │
│     - 「改正」「変更」キーワード検知     │
│                                         │
│  ③ GitHub Issue 自動起票                │
│     - 変更内容の要約を自動生成           │
│     - 影響を受ける問題の候補をリスト化   │
└─────────────────────────────────────────┘
```

**監視対象サイト:**
- https://www.shiken.or.jp/ （電気技術者試験センター）
- https://www.meti.go.jp/policy/safety_security/industrial_safety/ （経産省 産業保安）
- https://laws.e-gov.go.jp/law/409M50000400052 （電気設備技術基準 省令）

**実装:**
- GitHub Actions のスケジュール実行（毎日1回）
- 変更検知時に管理者へ即座に通知
- LLM で変更内容を要約し、影響範囲を推定

#### 第2層: 定期レビューサイクル（コスト: 中）

| タイミング | 作業内容 |
|---|---|
| **毎年1月** | 技能試験候補問題の確認・更新。試験センターが公表次第、候補問題コンテンツを更新 |
| **年2回（試験後）** | 新しい過去問を取得・収録。上期試験後（8月頃）と下期試験後（1月頃） |
| **年2回（3月・9月）** | 法規・技術基準の改正内容を総点検。影響を受ける問題・解説を修正 |
| **随時** | 自動監視で検出された変更への対応 |

#### 第3層: コミュニティ報告（コスト: ゼロ）

```
ユーザーが問題を解く
  ↓
「この問題の内容が古い？」ボタン
  ↓
報告がキューに溜まる
  ↓
管理者がレビュー → 修正or却下
```

- 各問題に「報告」ボタンを設置
- 「解説が間違っている」「法改正で答えが変わった」等を報告可能
- 報告数が多い問題は自動的にフラグが立つ
- 電気工事士資格保有者のコミュニティレビュアーを募集（将来）

### 技術的な対策: CMS設計

問題データをコードにハードコードしない。すべてDBで管理し、即座に更新可能にする。

```
questions テーブル:
  ├── valid_from: DATE      -- この問題が有効になる日
  ├── valid_until: DATE     -- この問題が無効になる日（法改正で陳腐化した場合）
  ├── revision: INT         -- 改訂回数
  ├── revision_note: TEXT   -- 「2025年技術基準改正に伴い選択肢を修正」等
  ├── is_active: BOOLEAN    -- 即座に出題停止できるフラグ
  └── legal_basis: TEXT     -- 根拠法令（「電技解釈第218条」等）
```

**運用フロー:**
1. 法改正を検知
2. `legal_basis` で影響を受ける問題を一括検索
3. 該当問題を `is_active = false` で即座に出題停止
4. 修正版を作成し `revision` をインクリメント
5. 修正版を `is_active = true` で公開
6. 旧版は `valid_until` を設定して履歴として保持

**アプリ側の対策:**
- オフラインキャッシュされた問題にもバージョン番号を付与
- アプリ起動時にサーバーとバージョンを照合
- 古い問題がキャッシュにある場合、バックグラウンドで差分更新
- 法改正で無効になった問題は「この問題は法改正により内容が変更されました」とユーザーに通知

### 年間運用コストの見積もり

| 項目 | コスト |
|---|---|
| 自動監視（GitHub Actions） | 無料（月2,000分以内） |
| LLMによる変更要約（Claude API） | 月500円程度 |
| 過去問追加（年2回、各50問） | 1回あたり8〜16時間の作業 |
| 法改正対応（年2〜3回） | 1回あたり4〜8時間の作業 |
| 技能試験候補問題更新（年1回） | 8〜16時間の作業 |
| **合計（個人運営の場合）** | **月1,000円 + 年間80〜120時間の作業** |

→ プレミアム課金ユーザーが10人以上いれば運用コストは回収可能
